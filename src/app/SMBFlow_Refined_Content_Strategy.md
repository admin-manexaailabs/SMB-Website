## Product Positioning Framework

### What is SMBFlow.ai?
SMBFlow.ai is an AI-powered workflow orchestration platform built on autonomous agents and LLM-driven reasoning. It automates end-to-end business processes across customer service, scheduling, campaigns, and operations — without requiring code or technical expertise.

### Core Product Definition
- **Platform Type**: Agentic AI automation platform
- **Target User**: Small and medium businesses (non-technical)
- **Architecture**: Multi-agent orchestration with LLM-based decision making
- **Deployment**: Multi-channel (WhatsApp, voice, SMS, email, web, Instagram, API)
- **Development**: Visual no-code builder with pre-built templates

---

## Problems SMBFlow.ai Addresses

### 1. Operational Bottlenecks
Small businesses manually handle repetitive tasks that AI agents can automate: answering FAQs, booking appointments, order tracking, sending reminders, qualifying leads.

### 2. Channel Fragmentation
Customers reach out via multiple channels (WhatsApp, phone, Instagram, email, SMS, web). Managing these separately is inefficient and error-prone.

### 3. After-Hours Coverage
SMBs lose revenue when inquiries arrive outside business hours. Agents provide 24/7 coverage without hiring additional staff.

### 4. Scaling Constraints
As businesses grow, conversation volume increases. Human-only teams don't scale. AI agents handle 10 to 10,000 conversations with the same infrastructure.

### 5. Technical Barriers
Building custom automation requires developers, APIs, and ongoing maintenance. SMBFlow.ai removes these barriers with a visual builder and pre-configured templates.

---

## How SMBFlow.ai Works (High-Level, Non-Technical)

### Step 1: Build Your Agent (Visual Workflow)
Use the drag-and-drop builder to design how your agent behaves:
- **Triggers**: When should the agent activate? (customer message, scheduled time, API event)
- **AI Reasoning**: What should the agent understand and decide?
- **Actions**: What should it do? (book appointment, fetch data, send message, create ticket)
- **Routing**: When should it escalate to a human?

### Step 2: Connect Your Data
Upload documents (menus, FAQs, price lists) or connect your existing systems:
- Shopify for product info
- MySQL/BigQuery for customer data
- Calendar systems for availability
- REST APIs for custom integrations

### Step 3: Deploy to Channels
Choose where your agent should operate:
- WhatsApp Business
- Voice calls (inbound/outbound)
- SMS
- Email
- Web chat widget
- Instagram DM
- Custom API endpoints

### Step 4: Monitor and Optimize
All conversations appear in a unified inbox. Review agent performance, see where humans intervene, and refine workflows based on real data.

## Key Features (Aligned with Design Sections)

### Agentic AI Core
- **Multi-step reasoning**: Agents understand context, plan actions, execute tasks
- **Intent classification**: Automatically detect what customers want
- **Dynamic decision trees**: Agents choose paths based on conversation context
- **Task execution**: Book, update, cancel, create, send — not just respond

### Visual Workflow Builder
- **Drag-and-drop interface**: No code required
- **Node types**: Triggers, prompts, conditions, actions, integrations, deployments
- **Real-time testing**: Simulate conversations before deploying
- **Version control**: Rollback to previous versions

### Multi-Channel Orchestration
- **Single source of truth**: One agent, deployed everywhere
- **Channel-aware formatting**: Agents adapt messages to each platform's rules
- **Unified conversation history**: See entire customer journey across channels
- **Channel-specific triggers**: Different entry points per channel

### Human-in-the-Loop
- **Smart escalation**: Agents know when to hand off to humans
- **Context transfer**: Full conversation history passed to human agents
- **Unified inbox**: All channels in one interface
- **Agent assist**: AI suggests responses even in human-handled chats

### Knowledge & Integrations
- **Document upload**: PDFs, CSVs, knowledge bases
- **Database connections**: MySQL, PostgreSQL, BigQuery, Supabase
- **E-commerce**: Shopify, WooCommerce, custom APIs
- **CRM/Support**: Zendesk, HubSpot, Salesforce
- **Calendar**: Google Calendar, Calendly, custom booking systems

### Multi-LLM Selection
- **Model choice per agent**: GPT-4, Claude 3, Gemini, Llama 3
- **Cost optimization**: Use cheaper models for simple tasks, advanced models for complex reasoning
- **Fallback logic**: Secondary model if primary fails
- **Custom prompts**: Fine-tune agent behavior per use case

### Analytics & Insights
- **Conversation volume**: Track total conversations, per channel, per agent
- **Resolution rate**: % handled by AI vs. escalated to humans
- **Response time**: Average time to first response and resolution
- **Task success rate**: % of bookings, orders, queries completed successfully
- **Cost per conversation**: Track operational savings

## Tone & Voice Guidelines

### Tone Attributes
- **Confident, not boastful**: "Agents execute multi-step tasks" vs. "Revolutionary AI"
- **Technical, not jargon-heavy**: "Agentic AI" is acceptable; "Synergistic orchestration" is not
- **Specific, not generic**: "Deploy to WhatsApp in 1 click" vs. "Easy deployment"
- **Business-focused, not tech-focused**: Lead with outcomes, not architecture

### Language Rules
✓ **Use**: Agents, workflows, automation, execute, deploy, orchestration, reasoning  
✗ **Avoid**: Revolutionary, game-changing, cutting-edge, seamless, powerful, innovative

✓ **Use**: "Agents reason, plan, and execute tasks"  
✗ **Avoid**: "Our AI-powered solution leverages advanced machine learning"

✓ **Use**: "85% cost reduction"  
✗ **Avoid**: "Massive savings"

✓ **Use**: "No code required"  
✗ **Avoid**: "Anyone can use it"

### Writing Style
- **Short sentences**: 15-20 words max for body copy
- **Active voice**: "Agents handle 1000 conversations" not "1000 conversations are handled"
- **Numbers over adjectives**: "93% accuracy" not "highly accurate"
- **Specific over vague**: "WhatsApp, voice, SMS" not "multiple channels"
